# SkillCODER 展示前端

纯 **HTML + CSS + 原生 JS**，无框架、无构建、无 npm——三个文件，队友打开即可读懂。

```
frontend/
├── index.html   # 页面骨架：① 背景 ② 案例 ③ 故事流 ④ 裁决 ⑤ 指标
├── styles.css   # 样式（含三大阶段配色）
├── app.js       # 全部逻辑：拉数据 + 按 visual.type 渲染 8 步 + 交互
├── config.js    # 后端地址配置
└── README.md
```

## 怎么跑（二选一）

### 方式 A：后端一并托管（推荐，一条命令）
后端已把本目录挂在 `/ui`。启动后端后直接访问：

```bash
cd ../backend && bash run.sh
# 浏览器打开： http://127.0.0.1:8000/ui/
```

### 方式 B：前端单独起一个静态服务器
```bash
cd frontend
python3 -m http.server 8080
# 浏览器打开： http://127.0.0.1:8080/
```
此方式下前端默认连 `http://127.0.0.1:8000` 的后端（后端已开 CORS）。
若后端在别处，用 URL 参数覆盖：`http://127.0.0.1:8080/?api=http://192.168.1.10:8000`

## 页面结构（对应总体设计文档）

| 区块 | 用到的接口 | 说明 |
|---|---|---|
| ① 背景 | 静态 | Skill 为何易被盗 + 两个技术难点 |
| ② 案例 | `GET /api/demo/cases` | 5 张卡片，点击进入演示 |
| ③ 故事流 | `GET /api/demo/cases/{case_id}` | 八步时间线，每步含 story + 可视化 + 折叠技术细节；支持上一步/下一步/自动播放 |
| ④ 裁决 | 复用③的 summary | 判决书卡：所有权 / 溯源 / 攻击 / 数据出处 |
| ⑤ 指标 | `GET /api/metrics/overview` | 论文聚合指标 + 本次真实运行实测 |

## 代码怎么读（app.js）

- `boot()`：探活后端 → 拉案例与指标。后端没起会显示友好提示。
- `selectCase()`：点案例 → 拉八步数据 → 渲染时间线与裁决。
- `VISUAL`：一个对象，8 个键对应后端 8 种 `visual.type`，每个键是一个渲染函数。
  **想改某一步长什么样，只改对应那一个函数即可。**
- `bitsGrid()`：把 0/1/⊥ 码字画成彩色方格（第 4/5/8 步共用）。

> 设计原则与后端一致：前端不懂算法，只把后端给的「story 大字 + visual 数据」照着画。

## 三大阶段配色

跟随后端 `step.phase` 字段自动着色：
- `embedding`（阶段一·水印嵌入）→ 蓝
- `deploy`（过渡·部署/攻击）→ 灰
- `verification`（阶段二·黑盒验证）→ 绿
